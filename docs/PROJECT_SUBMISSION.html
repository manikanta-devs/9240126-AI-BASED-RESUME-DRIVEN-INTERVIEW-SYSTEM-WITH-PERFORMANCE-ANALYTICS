Project Title and Objective

Title: AI-BASED RESUME DRIVEN INTERVIEW SYSTEM WITH PERFORMANCE ANALYTICS

Objective: Automatically generate personalized interview questions from a resume, evaluate text and voice answers with AI, and provide analytics and coaching to improve candidate performance.

Problem Statement

Generic interview practice does not tailor to candidate resumes and lacks measurable feedback; students need personalized practice and actionable analytics.

Literature Survey (short)

  • Resume parsing & NER: historical pipelines and modern transformer approaches.
  • Question generation: LLM prompt engineering vs. template-based fallbacks.
  • Automated evaluation: combine LLM scoring with deterministic heuristics for reliability.
  • Learning analytics: dashboards and skill-gap reports improve targeted practice.

Methodology / Approach

  1. Resume Analysis: extract skills/roles to seed prompts.
  2. Question Generation: AI-first prompts producing structured JSON; fallback bank used offline.
  3. Interview Flow: Frontend simulates interviews (text/voice); backend handles sessions and adaptation.
  4. Answer Evaluation: LLM-based when available; deterministic fallback heuristics otherwise.
  5. Analytics: aggregate session data into dashboards and skill-gap reports.

Dataset and Tools Used

  • Data: data/quizzes.json, data/sessions.json
  • Backend: Python 3.10+, Flask, requests, wikipedia
  • AI Providers: Hugging Face Inference, optional local LLM (transformers), Gemini fallback
  • Frontend: React + Vite + Tailwind (source in frontend/src/)
  • Dev tools: black, flake8, pytest, ESLint

Work Completed So Far

  • AI provider chain & JSON parsing: backend/ai/gemini_service.py
  • Question generator & fallback templates: backend/ai/question_generator.py
  • Answer evaluator: backend/ai/answer_evaluator.py
  • Frontend interview & analytics UI: frontend/src/pages/InterviewPage.jsx, frontend/src/pages/AnalyticsPage.jsx
  • Basic testing/linting and docs for handoff

Future Plan and Expected Outcomes

Short term: fix frontend lint issues, add unit tests and privacy docs. Medium term: add CI, improve evaluation robustness, and run a user study. Expected: personalized interviews, actionable analytics, and improved student performance.
